# 역할
너는 10년차 안드로이드 전문 개발자야.
프로젝트 구조화 및 설계에 경험이 많아.
AI 활용 프로젝트 제작 경험이 다수 있어.

## 배경
https://github.com/facebook/meta-wearables-dat-android/tree/main/samples/DisplayAccess
링크는 meta ray-ban DisplayAccess 샘플이야.
이 샘플을 이용해서 사용자가 음성으로 "근처 스타벅스 매장 알려줘" 처럼 특정 위치를 요청하면 구글맵을 이용해서 매장의 위도, 경도를 찾고 사용자가 해당 위치로 이동 할 수 있도록 인디캐이터 화살표를 띄워 방향을 알려주는 앱을 만들려고해.

## 요구사항
다음 내용이 반영 되도록 계획을 세워서 `docs/PLAN.md` 에 작성해줘. 
- STT로 사용자의 음성을 읽는다.
- STT로 인식된 문구는 화면 UI에 표시.
- 스트리밍 중에는 계속 리스닝하고 있어야 하고(TTS 재생 중엔 잠깐 멈췄다가 재개)
- 입력받은 STT 데이터의 장소 인식 분석은 앱이 OpenAI API를 직접 호출하여 구현한다.
- LLM API에서 분석한 위치 정보를 바탕으로 구글 Places API(New)를 이용해 목적지로 찾아 가는 안내 인디캐이터 화살표를 화면에 표시한다.
- 사용자의 gps, 자이로센서 등의 값을 이용해서 목적지를 안내하는 인디캐이터를 구현한다.
- 스마트글래스 화면에 저주기로 갱신되는 미니맵이 함께 표시된다.
- 위치 변화가 작을 때는 지도 재생성이 발생하지 않아 배터리와 전송 부하가 과하지 않게 한다.
- 지도 렌더링이 실패해도 내비게이션 안내 자체는 중단되지 않는다.
- 지도 소스는 `Google Maps Static API` 를 사용한다. 
- `Places API(New)`, `Google Maps Static API`는 하나의 api key 로 사용한다.
